# 整合包与服务器配置

这一篇面向整合包作者、服务器管理员和想调整 RTSBuilding 生存规则的玩家。

如果你只是正常游玩，通常只需要阅读“技能树与能力解锁”和“RTS 家园与范围限制”。

## 关键规则选项

按 `G` 打开 RTS 界面，进入齿轮设置面板，可以修改以下关键参数：

- “生存平衡”：是否启用技能树解锁。
- “队伍共享”：多人队伍是否共享解锁进度和家园锚点。
- “最大操作半径”：限制后期 RTS 能操作多远。
- “蓝图”：是否启用蓝图功能。
- “蓝图最大方块数”：限制单个蓝图导入、框选保存和建造的最大规模。
- “技能材料消耗”：自定义每个技能节点需要的材料。

多人游戏中，服务器配置会覆盖客户端本地设置。


## 自定义技能材料消耗

在“技能材料消耗”区域中，找到对应技能节点的材料输入框，直接填写材料列表。

材料格式是“物品 ID:数量”。多个材料用逗号分开：

```text
minecraft:chest:2,minecraft:redstone:8
minecraft:copper_ingot:16
minecraft:diamond_pickaxe:1,minecraft:redstone_block:1
```

点击 Reset，可以把当前技能节点恢复为默认材料。

保存后规则生效。多人游戏中由服务端配置决定。


## 调整解锁节奏

想让 RTSBuilding 更早进入基地建设阶段，可以降低“视角核心”“存储绑定”“远程放置”等早期节点的材料消耗。

想让 RTSBuilding 更像后期科技，可以提高关键节点材料，或者把“远程挖掘”“合成终端”“蓝图”相关能力放到更靠后的材料门槛。

调整时建议优先改材料消耗，而不是关闭大量功能。这样玩家仍然能看到完整能力树，只是需要按整合包节奏逐步解锁。
